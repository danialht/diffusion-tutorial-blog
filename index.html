2. DDPM Paper

Denoising diffusion probability models (DDPM) were introduced in a very famous paper by Ho et. al The goal for something like image generation is to be able to sample from a complicated distribution $p(x)$ such as the space of all realistic images or the space of all pictures of human faces. However, sampling from a distribution like this is very hard. In contrast, sampling from a normal distribution is super easy. So, can we start with a normal distribution (to get the randomness) and then work our way towards the desired distribution? The answer is yes! But how do we work our way back? The idea in this paper is that we learn the inverse of a markov chain that converts from $p(x)$ to the normal distribution.

2a. Forward Process in DDPM

A very easy markov chain that does this is one that that adds noise at each step like this one: $$X_{t+1} = \sqrt{\alpha_t}X_t + \sqrt{1-\alpha_t} Z_t \Longleftrightarrow q(x_{t+1}|x_t) = \mathcal N(\sqrt{\alpha_t}x_t, (1 - \alpha_t)I_d)$$ where $Z_t \sim \mathcal N(0, I_d)$. If you simulate a few steps, you can see that if you define $\bar\alpha_t = \prod_{s = 1}^t \alpha_s$, then $$X_t \sim \mathcal N(\sqrt{\bar\alpha_t} X_0, (1 - \bar\alpha_t) I_d)$$ Therefore, after adding enough noise, after a certain time $t$, $X_t$ will effectively be distributed as a unit normal distribution. Here's what that process looks like for the smiley face distribution.

3. Score Matching

3a. Denoising Score Matching

For every distribution $p$, the score function is defined as $s(x) = \nabla_x \log p(x)$. At any point $x$, it is a vector pointing in the direction of steepest increase in log-density, in other words, toward where the distribution puts more of its probability mass i.e. region that has higher probability. Note that, score function's direction is the same as the gradient of the probability density function, since $\nabla \log p(x) = \frac{\nabla p(x)}{p(x)}$. Another nice property of the score function is that you can compute without having the probability density, having the unnormalized density would suffice, i.e. for any function $\tilde{p} \propto p$ we have: \[ \nabla \log p(x) = \frac{\nabla p(x)}{p(x)} = \frac{\nabla \tilde{p}(x)}{\tilde{p}(x)} = \nabla \log \tilde{p} (x)\] Last point that it really important for us, knowing the score is enough to draw samples from the distribution since there is an elegant algorithm for this called Langevin dynamics. Here is the algorithm pseudocode: 
  \begin{algorithm}
  \caption{Langevin Dynamics Sampling}
  \begin{algorithmic}
  \State Sample $X$ from any distribution
  \For{$i = 1, 2, \ldots, K$}
    \State $z_i \sim \mathcal{N}(0, I)$
    \State $X \leftarrow X + \frac{\epsilon}{2} \nabla \log p(X) + \sqrt{\epsilon}\, z_i$
  \EndFor
  \Return $X$
  \end{algorithmic}
  \end{algorithm}
And here is a Python code implementation for the case where $p(x)$ is a mixture of two gaussians, one $\mathcal{N}(\pm \mu, \sigma^2)$, where the mixture weights each of them equally. The score function in this case is: $$ \frac{d}{dx} \log p(x) = \frac{\frac{d}{dx}p(x)}{p(x)} = \frac{1}{\sigma^2} (-x + \mu \tanh\left(\frac{\mu x}{\sigma^2}\right)) $$ 
import numpy as np

N = 10_000  # num samples
K = 100 # num iterations
EPSILON = 0.05 # step size

x = 4*(np.random.rand(N)-0.5)  # initial distribution = Uniform(-2, 2)
# score function for a mixture of N(-mu, sigma^2) and N(mu, sigma^2)
mu, sigma = 2, 1.0
score_fn = lambda x: (1/sigma**2) * (-x+mu*np.tanh(x*mu/sigma**2))

for _ in range(K):
    z = np.random.randn(N)
    x += EPSILON/2 * score_fn(x) + EPSILON**.5 * z
Training a model to learn the score function is called score matching. Rather than learning the likelihood $p(x)$ directly, we only learn its gradient, which turns out to be enough for sampling. The catch is that we never actually have $p(x)$. All we have is a dataset of samples, which represents a spiky empirical distribution, positive at the data points and zero everywhere else. The score of something that spiky is not useful. So we smooth it out first. The smoothed distribution $p_\sigma(x)$ is simple to define: draw a sample $x$ from your dataset, then add Gaussian noise $\sigma z$. The result is a sample from $p_\sigma$. Geometrically, this is equivalent to replacing each data point with a Gaussian of width $\sigma$ centered on it, so you get a mixture of $N$ Gaussians, and you estimate the score of $p$ with the score of that mixture. This raises a natural question: how much noise should you add? Too little, and the score is only well-estimated near the data points, useless when sampling starts far away. Too much, and the score points toward a blob that no longer resembles the data. The answer is that you need both: use a large $\sigma$ when you are far from the data, and a small $\sigma$ when you are close. This leads to the idea of training a single score network conditioned on the noise level: \[ s_\theta(\sigma, x) \approx \nabla \log p_\sigma(x) \] 
  \begin{algorithm}
  \caption{Annealed Langevin Dynamics}
  \begin{algorithmic}
  \Require noise schedule $\sigma_1 < \sigma_2 < \cdots < \sigma_K$, score network $s_\theta$, step sizes $\{\epsilon_i\}$, steps per level $T$
  \State Sample $X \sim \pi(x)$ \Comment{any initial distribution}
  \For{$i = K, K-1, \ldots, 1$}
    \For{$t = 1, 2, \ldots, T$}
      \State $z \sim \mathcal{N}(0, I)$
      \State $X \leftarrow X + \frac{\epsilon_i}{2} s_\theta(X, \sigma_i) + \sqrt{\epsilon_i}\, z$
    \EndFor
  \EndFor
  \Return $X$
  \end{algorithmic}
  \end{algorithm}

3b. Introduction to SDEs

Now let's start viewing this through another lens, Langevin dynamics incentivized us to learn the score function and use that for sampling. Now we will focus more on the forward process and will introduce a new notion. Assume a family of random variables $\{X_t\}_{t \in [0, T]}$ such that $X_t \sim p_t(x)$. Each $X_t$ has a different distribution but changing $t$ slightly will not disturb the distribution a lot, i.e. distribution of $X_t$ and $X_{t+\delta t}$ are the same in the limit $\delta t \rightarrow 0$. This is a good thing, since now you can sample from $X_t$ and perturb the samples a bit and have some samples from $X_{t+\delta t}$, the perturbation could be deterministic or stochastic, like moving all the samples in a direction for length $a\delta t$, or adding a gaussian with variance $\sigma^2 \delta t$. We can describe this change of the distribution of $X_t$ that happens because of the change in $t$ with a SDE (Stochastic Differential Equation). Here is a general form of a SDE: \[dX_t = f(X_t, t) dt + g(t) dW_t\] The first term is called the drift and the second term is called the diffusion. For every $t$ this SDE defines a relation between the distribution of $X_t$ and $X_{t+\delta t}$ for $\delta t \rightarrow 0$ in this way: In order to sample from the distribution $X_{t+\delta t}$ you can sample from $X_t$ and then add a deterministic perturbation term to the sample with that is $f(X_t, t)dt$ and then on top of that add a stochastic perturbation $g(t) dW$ where $g(t)$ is a scalar and $dW$ is a gaussian with variance $dt$. Now let's look at some examples of SDEs:
Example 1:
\[ dX_t = \sigma(t) dW_t \] this SDE keeps adding gaussian noise to $X_t$ as $t$ grows. Let's say that the initial distribution is $\delta(0)$, meaning that $X_0=0$ with probability 1, then what would be the distribution of $X_t$? Well, $X_t$ is $X_0 = 0$ plus many small gaussians that are independent from each other. This means that $X_t$ is also Gaussian. Therefore, we can add up the means of the differential Gaussians that were added to make find the mean of $X_t$. In order to find the variance we could add up the variance of the differential Gaussians since they are independent and thus we have: \[\mathbb{E}[X_t] = 0, \quad \text{Var}[X_t] = \int_0^t \sigma^2(s) ds\] \[\Rightarrow X_t \sim \mathcal{N}(0, \int_0^t \sigma^2(s) ds) \] Example 2:
\[ dX_t = -\frac{\beta}{2} X_t \, dt + \sqrt{\beta} \, dW_t \] with a constant $\beta > 0$ chosen large (say $\beta = 10$). Let the initial distribution be a mixture of Gaussians: \[ X_0 \sim \sum_k w_k \,\mathcal{N}(\mu_k, \sigma_k^2) \] We will show that although $X_0$ is a mixture of Gaussians, $X_1 \approx \mathcal{N}(0,1)$.

Let's solve this SDE. The drift term $-\frac{\beta}{2} X_t \, dt$ makes this harder than Example 1 because the noise added at each step gets multiplied by the exponential decay before reaching time $t$. The standard trick is an integrating factor. Let $Y_t = e^{\beta t/2} X_t$. (Usually we need Itô's calculus to deal with derivatives in SDEs, but in this example we can get away with just using regular calculus): \[ dY_t = \frac{\beta}{2} e^{\beta t/2} X_t \, dt + e^{\beta t/2} \, dX_t = e^{\beta t/2}\sqrt{\beta} \, dW_t \] (the drift terms cancel). This is exactly Example 1's SDE for $Y_t$, so we can integrate directly: \[ Y_t = Y_0 + \sqrt{\beta} \textcolor{blue}{\int_0^t e^{\beta s/2} \, dW_s} \] In order to simplify the blue term, we know that since it is taking an integral over differential Gaussians, then it is some Gaussian. Thus: \[ \int_0^t e^{\beta s/2} \, dW_s = \sigma(t) Z \quad \text{where } Z \sim \mathcal{N}(0, 1) \] Now, in order to find $\sigma(t)$, we just need to take the variance of both sides, also note that when we have a set of i.i.d. random variables, the variance of sum is the sum of variances, thus: \[ \text{Var}\left[\int_0^t e^{\beta s/2} \, dW_s\right] = \text{Var}[\sigma(t)Z] \Rightarrow \int_0^t e^{\beta s} \, \text{Var}[dW_s] = \sigma^2(t) \] \[ \Rightarrow \sigma^2(t) = \int_0^t e^{\beta s} \, ds = \frac{1}{\beta}( e^{\beta t} - 1 ) \] \[ \Rightarrow Y_t = Y_0 + \sqrt{e^{\beta t} - 1} \, Z \quad \text{where } Z \sim \mathcal{N}(0, 1) \] Substituting back $X_t = e^{-\beta t/2} Y_t$: \[ X_t = e^{-\beta t/2} X_0 + \sqrt{1 - e^{-\beta t}} \, Z \] Therefore: \[ X_t \mid X_0 \;\sim\; \mathcal{N}\!\left(e^{-\beta t/2} X_0,\; 1 - e^{-\beta t}\right) \] we could marginalize over $X_0$, $X_0$ is a sampled from $\mathcal{N}(\mu_k, \sigma_k^2)$ with probability $w_k$: \[ p_t(x) = \sum_k w_k \,\mathcal{N}\!\left(x;\; e^{-\beta t/2}\mu_k,\; e^{-\beta t}\sigma_k^2 + (1 - e^{-\beta t})\right) \] At $t=0$. Plug in $t=0$: $e^0 = 1$, so the mean of each component is $\mu_k$ and the variance is $\sigma_k^2 + 0 = \sigma_k^2$. This recovers the original mixture exactly.
At $t=1$. For large $\beta$, $e^{-\beta} \approx 0$. Every component in the mixture has: \[ \text{mean} \approx e^{-\beta/2} \mu_k \approx 0, \qquad \text{variance} \approx e^{-\beta}\sigma_k^2 + 1 \approx 1 \] All $K$ components have collapsed to essentially $\mathcal{N}(0,1)$, regardless of their original means and variances. Since $\sum_k w_k = 1$: \[ p_1 \approx \mathcal{N}(0, 1) \quad \] This is the key mechanism behind diffusion models: the drift pulls everything toward the origin while the noise inflates the variance toward $1$, and these two effects conspire to erase the structure of the original distribution.
Example 3:
Now we will introduce an important fundamental theorem about SDEs.
Theorem (Anderson, 1982). Let $\{X_t\}_{t \in [0,T]}$ satisfy the forward SDE \[ dX_t = f(X_t, t)\, dt + g(t)\, dW_t \] with marginal densities $\{p_t\}$. Then the time-reversed process $\{\tilde X_t\}_{t \in [0,T]}$, defined by running time backwards from $T$ to $0$, satisfies the reverse SDE \[ d\tilde X_t = \bigl[f(\tilde X_t, t) - g(t)^2 \,\nabla_x \log p_t(\tilde X_t)\bigr]\, dt + g(t)\, d\bar W_t \] where $d\bar W_t$ is a Brownian motion running backwards in time, and $\tilde X_T \sim p_T$.
Applying the theorem to Example 2. By plugging $f(x,t) = -\frac{\beta}{2}x$ and $g(t) = \sqrt{\beta}$ into the reverse SDE, we get: \[ d\tilde X_t = \left[-\frac{\beta}{2}\tilde X_t - \beta\, \nabla_x \log p_t(\tilde X_t)\right] dt + \sqrt{\beta}\, d\bar W_t \] where $p_t$ is the mixture-of-Gaussians marginal we derived in Example 2. If we initialize $\tilde X_1 \sim \mathcal{N}(0,1)$ and run this SDE backwards from $t=1$ to $t=0$, the distribution at $t=0$ will be (approximately) the original mixture of Gaussians. The proof is left to the reader as an exercise. The "inverse SDE" and "inverse time" simply mean we run from $t=1$ to $t=0$, so $dt < 0$ — without this sign flip, the dynamics would push samples away from the target rather than toward it. An equivalent formulation substitutes $\tau = T - t$, turning the reverse SDE into a standard forward SDE in $\tau \in [0, T]$ with Wiener increment $dW_\tau$.
Using the theorem we just mentioned, we can now use score matching and learn the score, after that in order to sample from the target distribution we can just run a discretized version of the reverse SDE and that's it! We will now talk about different methods of discretizing and sampling the SDE.
