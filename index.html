2. DDPM Paper

Denoising diffusion probability models (DDPM) were introduced in a very famous paper by Ho et. al The goal for something like image generation is to be able to sample from a complicated distribution $p(x)$ such as the space of all realistic images or the space of all pictures of human faces. However, sampling from a distribution like this is very hard. In contrast, sampling from a normal distribution is super easy. So, can we start with a normal distribution (to get the randomness) and then work our way towards the desired distribution? The answer is yes! But how do we work our way back? The idea in this paper is that we learn the inverse of a markov chain that converts from $p(x)$ to the normal distribution.

2a. Forward Process in DDPM

A very easy markov chain that does this is one that that adds noise at each step like this one: $$X_{t+1} = \sqrt{\alpha_t}X_t + \sqrt{1-\alpha_t} Z_t \Longleftrightarrow q(x_{t+1}|x_t) = \mathcal N(\sqrt{\alpha_t}x_t, (1 - \alpha_t)I_d)$$ where $Z_t \sim \mathcal N(0, I_d)$. If you simulate a few steps, you can see that if you define $\bar\alpha_t = \prod_{s = 1}^t \alpha_s$, then $$X_t \sim \mathcal N(\sqrt{\bar\alpha_t} X_0, (1 - \bar\alpha_t) I_d)$$ Therefore, after adding enough noise, after a certain time $t$, $X_t$ will effectively be distributed as a unit normal distribution. Denoising Score Matching

Score: For every distribution $p$ the score function is defined as $s(x) = \nabla_x \log p(x)$. It points in the direction of higher density. Learning the score function is an implicit way to learn the distribution. The score function integrates to the log likelihood, but more importantly, using the score function of some distribution there is a genius way to sample from that distirbution, using Langevin dynamics. Here is the algorithm: 
  \begin{algorithm}
  \caption{Langevin Dynamics Sampling}
  \begin{algorithmic}
  \State Sample $X$ from any distribution
  \For{$i = 1, 2, \ldots, K$}
    \State $z_i \sim \mathcal{N}(0, I)$
    \State $X \leftarrow X + \frac{\epsilon}{2} \nabla \log p(X) + \sqrt{\epsilon}\, z_i$
  \EndFor
  \Return $X$
  \end{algorithmic}
  \end{algorithm}
  
import numpy as np

N = 10_000  # num samples
K = 100
EPSILON = 0.1

x = 4*(np.random.rand(N)-0.5)  # initial distribution = Uniform(-2, 2)
mu, sigma = 1, 1.5
score_fn = lambda x: -(x - mu) / sigma**2

for _ in range(K):
    z = np.random.randn(N)
    x += EPSILON/2 * score_fn(x) + EPSILON**.5 * z
learning the score function is called score matching. It is an implicit method to learn the distirbution, meaning that we are not going to learn the likelihood directly. There is an obstacle when it comes to learning the score function, and that is that we don't have it, we are usually given a dataset that has datapoints, in order to have the score function we need to have the distribution itself. The datapoints that we have represent some empirical distribution that is spiky in some points and zero everywhere else, in order to get the score function we are going to smooth out this distribution. smoothing out means that instead of looking at the distribution $p(x)$ we are going to consider the distribution $p_\sigma(x)$. The noisy distribution is defined like this: Sample from the distribution $p(x)$ and then add noise $\sigma z$ where $z$ is gaussian, then you have a sample from the noisy distribution. A way to think of this perturbing, noisifying or smoothing out is that, we are estimating the distribution (that we have N samples from) with a with a mixture of N gaussians around these N samples, and we are estimating the score of the distribution that the N samples came from with the score of this mixture of Gaussians. There are a couple of questions: How much noise should we add? If we want to estimate the score somewhere that is far from the samples, should we use a lot of noise, or a small noise? what if we want to estimate the score somewhere that is close to the samples. we need large noise when we are estimating the scoer far from the samples and we need small noise if we are close to the samples. So a proposed method for sampling is this: train a score network to learn the score for each noisy distribution, $s_{\theta}(\sigma, x) \approx \nabla \log p_\sigma(x)$ 
\begin{algorithm}
\caption{Annealed Langevin Dynamics}
\begin{algorithmic}
\Require noise schedule $\sigma_1 < \sigma_2 < \cdots < \sigma_K$, score network $s_\theta$, step sizes $\{\epsilon_i\}$, steps per level $T$
\State Sample $X \sim \pi(x)$ \Comment{any initial distribution}
\For{$i = K, K-1, \ldots, 1$}
  \For{$t = 1, 2, \ldots, T$}
    \State $z \sim \mathcal{N}(0, I)$
    \State $X \leftarrow X + \frac{\epsilon_i}{2} s_\theta(X, \sigma_i) + \sqrt{\epsilon_i}\, z$
  \EndFor
\EndFor
\Return $X$
\end{algorithmic}
\end{algorithm}
